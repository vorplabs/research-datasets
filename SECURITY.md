# Security policy

Do not post credentials, private data, exploit details, or other sensitive material in a public issue.

Use the [Vorp Labs contact form](https://vorplabs.com/contact) and begin the message with **Security report**. Include the affected file or URL, impact, reproduction steps, and a safe way for Vorp Labs Research to follow up.

Data errors and stale sources are not security issues. Report those through [CORRECTIONS.md](CORRECTIONS.md).
